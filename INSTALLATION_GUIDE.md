# คู่มือการย้ายโปรเจคและติดตั้งเครื่องอื่น (Deployment & Installation Guide)

โปรเจคนี้พัฒนาโดยใช้โครงสร้างสถาปัตยกรรมแบบ **FastAPI (Backend)** และ **Vite (Frontend)** มีการเชื่อมโยงกับฐานข้อมูล **SQLite** และใช้ **Playwright** สำหรับการส่งออกรายงาน (Export PDF/JPG)

ท่านสามารถคัดลอกไฟล์และโฟลเดอร์ในโปรเจคนี้ไปรันหรือติดตั้งบนเครื่องคอมพิวเตอร์เครื่องอื่นได้อย่างแน่นอน โดยปฏิบัติตามขั้นตอนดังต่อไปนี้:

---

## 1. สิ่งที่ต้องเตรียมในเครื่องคอมพิวเตอร์ใหม่ (Prerequisites)

ก่อนทำการติดตั้ง โปรดตรวจสอบว่าเครื่องใหม่มีโปรแกรมเหล่านี้นี้ติดตั้งอยู่:
1. **Python** (แนะนำเวอร์ชัน 3.10 ขึ้นไป)
   * ดาวน์โหลดได้จาก: [python.org](https://www.python.org/)
   * ตอนติดตั้งโปรดติ๊กถูกที่ **"Add Python to PATH"** ด้วยทุกครั้ง
2. **Node.js** (แนะนำเวอร์ชัน LTS เช่น 18 หรือ 20 ขึ้นไป)
   * ดาวน์โหลดได้จาก: [nodejs.org](https://nodejs.org/)
3. **Web Browser (Google Chrome หรือ Microsoft Edge)** 
   * ใช้สำหรับการทำงานร่วมกับระบบ Export PDF/JPG ของ Playwright

---

## 2. วิธีการเตรียมไฟล์เพื่อย้ายเครื่อง (Preparing Files)

เมื่อทำการคัดลอกโฟลเดอร์โปรเจค **ไม่จำเป็น** ต้องคัดลอกโฟลเดอร์ที่เกิดจากการติดตั้งที่เครื่องเก่า (เนื่องจากมีความสำคัญสัมพันธ์กับ OS และมีขนาดใหญ่มาก)

### โฟลเดอร์/ไฟล์ที่สามารถข้าม (ไม่ต้องคัดลอกไป):
* ❌ `node_modules` (โฟลเดอร์ไลบรารีของฝั่งหน้าบ้าน)
* ❌ `venv` หรือ `.venv` (โฟลเดอร์ Python Virtual Environment ของเดิม)
* ❌ โฟลเดอร์ `dist` (ถ้ามี)

### โฟลเดอร์/ไฟล์หลักที่จำเป็นต้องคัดลอกไป:
*  `backend/` (โค้ดควบคุมระบบหลังบ้าน)
*  `frontend/` (โค้ดแสดงผลหน้าบ้าน)
*  `data/` (โฟลเดอร์เก็บไฟล์นำเข้า, ฐานข้อมูล SQLite `ofi_data.db` และการตั้งค่า)
*  `package.json` & `package-lock.json`
*  `requirements.txt`
*  `vite.config.js`
*  `tailwind.config.js` & `postcss.config.js`

---

## 3. ขั้นตอนการติดตั้งบนเครื่องใหม่ (Installation Steps)

เมื่อคัดลอกไฟล์โปรเจคไปวางที่เครื่องใหม่แล้ว ให้เปิด Command Prompt (CMD) หรือ PowerShell แล้วเข้าไปยังโฟลเดอร์ของโปรเจค จากนั้นสั่งงานตามขั้นตอนต่อไปนี้:

### ขั้นตอนที่ 3.1: ติดตั้งฝั่งหลังบ้าน (Backend Setup)

1. **สร้างสภาพแวดล้อมจำลอง (Virtual Environment):**
   ```bash
   python -m venv venv
   ```

2. **เปิดใช้งาน Virtual Environment (Activate):**
   * **สำหรับ Windows (PowerShell):**
     ```powershell
     .\venv\Scripts\Activate.ps1
     ```
   * **สำหรับ Windows (CMD):**
     ```cmd
     .\venv\Scripts\activate.bat
     ```
   * **สำหรับ macOS / Linux:**
     ```bash
     source venv/bin/activate
     ```

3. **ติดตั้งแพ็กเกจ Python ที่เกี่ยวข้อง:**
   ```bash
   pip install -r requirements.txt
   ```

4. **ติดตั้งและตั้งค่าเบราว์เซอร์ของ Playwright:** (จำเป็นสำหรับการส่งออกรายงาน)
   ```bash
   playwright install
   ```

### ขั้นตอนที่ 3.2: ติดตั้งฝั่งหน้าบ้าน (Frontend Setup)

1. **เปิด Command Prompt อีกหน้าต่างขึ้นมา** (ในโฟลเดอร์โปรเจค)
2. **ติดตั้งไลบรารีที่จำเป็นผ่าน npm:**
   ```bash
   npm install
   ```

## 4. วิธีการเปิดใช้งานระบบ (Running the Project)

เมื่อตั้งค่าเครื่องใหม่สำเร็จแล้ว ทุกครั้งที่ต้องการเปิดรันระบบ ให้ดำเนินการดังนี้:

### 1. สตาร์ทระบบหลังบ้าน (Backend Server - FastAPI)
เปิดใช้งาน `venv` แล้วรันคำสั่ง:
```bash
uvicorn backend.main:app --host 127.0.0.1 --port 8000 --reload
```
*(ระบบหลังบ้านจะทำงานที่ตำแหน่ง `http://localhost:8000`)*

### 2. สตาร์ทระบบหน้าบ้าน (Frontend Server - Vite)
เปิด Command Prompt อีกอันในโฟลเดอร์โปรเจค แล้วรันคำสั่ง:
```bash
npm run dev
```
*(ระบบจะแจ้งลิงก์สำหรับเข้าใช้งาน เช่น `http://localhost:5173/` หรือพอร์ตใกล้เคียง)*

เมื่อเปิดรันทั้งคู่แล้ว ท่านสามารถเปิดเบราว์เซอร์และเข้าไปที่ลิงก์ฝั่งหน้าบ้าน (Vite) เพื่อใช้งานระบบแดชบอร์ดทั้งหมดได้ทันที

## 5. การแก้ไขปัญหาเบื้องต้น (Troubleshooting)

* **ปัญหา: บีบอัดไฟล์แล้วติดสิทธิ์ Permission หรือไฟล์ขนาดใหญ่เกินไป**
  * ให้ลบโฟลเดอร์ `venv` และ `node_modules` ในเครื่องเก่าออกก่อนทำการบีบอัดไฟล์ (Zip) แล้วจึงนำไปคลายซิปบนเครื่องใหม่เพื่อติดตั้งตามขั้นตอนข้อ 3
* **ปัญหา: ระบบหลังบ้านรันขึ้นมาแล้วดับ หรือมีข้อผิดพลาดเกี่ยวกับ Unicode/Charmap**
  * ตรวจสอบว่าในไฟล์โค้ดของหลังบ้าน (`backend/services.py` และ `backend/main.py`) ไม่มีฟังก์ชันการพิมพ์ (Print) ที่มีอีโมจิพิเศษที่ CMD ของ Windows เครื่องใหม่ไม่รองรับ (เราได้ปรับปรุงเป็น Plain text เกือบทั้งหมดแล้วเพื่อความปลอดภัยสูงสุด)
